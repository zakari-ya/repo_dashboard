# Checklist de revue du projet

Avant de considérer une tâche comme terminée, vérifier les points suivants.

## Contraintes techniques

* [ ] Le projet utilise Next.js.
* [ ] Le projet utilise React.
* [ ] Le projet utilise TypeScript.
* [ ] Le projet utilise Tailwind CSS 4.
* [ ] Aucune API réelle n’est utilisée.
* [ ] Aucun back-end n’a été créé.
* [ ] Aucune route API Next.js n’a été créée.
* [ ] Aucune base de données n’est utilisée.
* [ ] Les données sont simulées localement.

## Interface

* [ ] Toute l’interface est en français.
* [ ] Le design est propre et professionnel.
* [ ] Le design ne ressemble pas à une interface générée par IA.
* [ ] La hiérarchie visuelle est claire.
* [ ] Les fichiers critiques sont visibles rapidement.
* [ ] Les badges de criticité sont cohérents.
* [ ] Les états vides sont prévus.

## Responsive design

* [ ] L’application est utilisable sur mobile.
* [ ] L’application est agréable sur desktop.
* [ ] Les filtres restent simples sur petit écran.
* [ ] La liste des fichiers reste lisible sur mobile.
* [ ] Les espacements sont réguliers.

## Code

* [ ] Le code est simple et lisible.
* [ ] Les composants sont réutilisables.
* [ ] La logique métier est séparée dans `src/lib`.
* [ ] Les types sont définis dans `src/types`.
* [ ] Aucun `any` inutile n’est utilisé.
* [ ] Aucun fichier inutile n’a été ajouté.
* [ ] Aucune dépendance inutile n’a été ajoutée.

## Fonctionnalités

* [ ] La liste des repositories s’affiche.
* [ ] La page détail d’un repository fonctionne.
* [ ] Les fichiers associés s’affichent.
* [ ] La recherche par nom fonctionne.
* [ ] Le filtre par langage fonctionne.
* [ ] Le filtre par criticité fonctionne.
* [ ] Le résumé du repository fonctionne.
* [ ] Le niveau global de risque est affiché.

## Documentation

* [ ] Le README est en français.
* [ ] Le README respecte les 14 sections demandées.
* [ ] La partie sur l’usage de l’IA est claire.
* [ ] La documentation d’installation est simple.
* [ ] Le lien de la vidéo de démonstration est prévu.
